Required Salesforce custom fields — one-time admin setup

Two objects need custom fields before any Apex code runs. All fields ending in __c are custom — they don't exist by default and must be created in Setup → Object Manager → [Object] → Fields & Relationships → New.

User object — agent identity mapping
These fields store the resolved mapping between Genesys and Gong per agent. Once mapped, no external API calls are needed for that agent on subsequent runs.

Phase 1 · Step 1 — Salesforce calls Genesys to retrieve call data
1
Scheduled Apex calls Genesys Analytics API and Recordings API
Named Credential: Genesys · Apex Callout
Genesys API

The Scheduled Apex job calls the Genesys Analytics API to fetch completed voice calls in the last time window, then calls the Recordings API per call to get the audio file URL.

// Phase1UploadJob.apex — Scheduled Apex entry point global class Phase1UploadJob implements Schedulable { global void execute(SchedulableContext sc) { Phase1Processor.run(); } } public class Phase1Processor { public static void run() { // Step 1a: call Genesys Analytics API String genesysToken = GenesysAuth.getToken(); String interval = buildInterval(); // last 1 hour ISO window String body = '{"interval":"' + interval + '","order":"asc","orderBy":"conversationStart",' + '"paging":{"pageSize":20,"pageNumber":1},' + '"segmentFilters":[{"type":"and","predicates":[{"type":"dimension",' + '"dimension":"mediaType","operator":"matches","value":"voice"}]}]}'; HttpRequest req = new HttpRequest(); req.setEndpoint('callout:Genesys_NC/api/v2/analytics/conversations/details/query'); req.setMethod('POST'); req.setHeader('Content-Type', 'application/json'); req.setHeader('Authorization', 'Bearer ' + genesysToken); req.setBody(body); HttpResponse res = new Http().send(req); Map<String,Object> parsed = (Map<String,Object>) JSON.deserializeUntyped(res.getBody()); List<Object> conversations = (List<Object>) parsed.get('conversations'); for (Object c : conversations) { Map<String,Object> conv = (Map<String,Object>) c; processConversation(conv, genesysToken); } } }
Named Credential: Genesys_NC must be configured in Salesforce Setup → Named Credentials with the Genesys OAuth 2.0 client credentials. The callout URL then becomes callout:Genesys_NC/api/v2/... — no hardcoded URLs or tokens in Apex.
Phase 1 · Step 2 — SOQL Task lookup via clientUniqueId
2
SOQL query on Task object → follow WhoId chain to Account + Opportunity
Internal Salesforce · no API call needed
SOQL

Using the clientUniqueId from the Genesys call, Salesforce queries its own Task table directly. No REST API required — this is a native SOQL query inside the same Apex transaction.

// Step 2: find Task by clientUniqueId, follow chain to Account + Opportunity String clientUniqueId = '3b10d6d6-13be-abcd'; // from Genesys participant attributes List<Task> tasks = [ SELECT Id, WhoId, Who.AccountId FROM Task WHERE clientUniqueId__c = :clientUniqueId LIMIT 1 ]; if (tasks.isEmpty()) { // No matching Task — log and skip this call logError(clientUniqueId, 'No Task found for clientUniqueId'); return; } Task t = tasks[0]; Id contactId = t.WhoId; // e.g. 003XXX Id taskId = t.Id; // e.g. 00TAAA — stored for Phase 2 // Follow chain: Contact → Account → open Opportunity List<Contact> contacts = [ SELECT Id, AccountId, Account.Name, (SELECT Id, Name, StageName FROM Opportunities WHERE IsClosed = false ORDER BY LastModifiedDate DESC LIMIT 1) FROM Contact WHERE Id = :contactId LIMIT 1 ]; Contact con = contacts[0]; Id accountId = con.AccountId; Id opportunityId = con.Opportunities.isEmpty() ? null : con.Opportunities[0].Id; // These IDs go into the Gong context block in Step 4
Phase 1 · Step 3 — Resolve Genesys User ID to Gong Numeric ID
How mapping works: Salesforce uses the User object as the mapping store. Once an agent's Gong Numeric ID is resolved and saved, it is reused on all future calls — no repeated API calls for already-mapped agents.

Case A — Already mapped

Salesforce User has both Genesys_User_Id__c and Gong_Numeric_Id__c populated. Use the Gong ID directly. Zero external API calls.

Case B — Genesys ID on User, no Gong ID

User has Genesys_User_Id__c set but Gong_Numeric_Id__c is blank. Use the User's standard Email to query Gong GET /v2/users, save the result to the User record.

Case C — No mapping at all

Genesys User ID not yet on any Salesforce User. Call Genesys GET /api/v2/users/{id} to get email → match to SF User → call Gong to get Numeric ID → save both to User record.

// Step 3: resolve Genesys userId → Gong Numeric Id // Uses SF User record as the mapping store — avoids repeated API calls public static String resolveGongUserId(String genesysUserId, String genesysToken) { // Case A: check if SF User already has both IDs mapped List<User> users = [ SELECT Id, Email, Gong_Numeric_Id__c, Gong_Mapping_Status__c FROM User WHERE Genesys_User_Id__c = :genesysUserId LIMIT 1 ]; if (!users.isEmpty() && users[0].Gong_Numeric_Id__c != null) { // Case A: already mapped — return immediately return users[0].Gong_Numeric_Id__c; } String agentEmail; User sfUser; if (!users.isEmpty()) { // Case B: SF User found by Genesys ID but no Gong ID yet — use their email sfUser = users[0]; agentEmail = sfUser.Email; } else { // Case C: no SF User mapped — call Genesys Users API to get email HttpRequest req = new HttpRequest(); req.setEndpoint('callout:Genesys_NC/api/v2/users/' + genesysUserId); req.setMethod('GET'); req.setHeader('Authorization', 'Bearer ' + genesysToken); HttpResponse res = new Http().send(req); Map<String,Object> genUser = (Map<String,Object>) JSON.deserializeUntyped(res.getBody()); agentEmail = (String) genUser.get('email'); // Find the matching SF User by email and write the Genesys ID List<User> matched = [SELECT Id, Email FROM User WHERE Email = :agentEmail LIMIT 1]; if (matched.isEmpty()) { return null; } // no SF User — cannot proceed sfUser = matched[0]; sfUser.Genesys_User_Id__c = genesysUserId; } // Case B + C: query Gong /v2/users to find numeric ID by email String gongId = lookupGongIdByEmail(agentEmail); if (gongId == null) { sfUser.Gong_Mapping_Status__c = 'Error'; sfUser.Gong_Mapping_Error__c = 'No Gong user found for email: ' + agentEmail; } else { sfUser.Gong_Numeric_Id__c = gongId; sfUser.Gong_Mapping_Status__c = 'Mapped'; sfUser.Gong_Mapped_At__c = Datetime.now(); } update sfUser; // persist mapping for future calls return gongId; } // Helper: call Gong GET /v2/users and match by email private static String lookupGongIdByEmail(String email) { HttpRequest req = new HttpRequest(); req.setEndpoint('callout:Gong_NC/v2/users'); req.setMethod('GET'); HttpResponse res = new Http().send(req); Map<String,Object> data = (Map<String,Object>) JSON.deserializeUntyped(res.getBody()); List<Object> users = (List<Object>) data.get('users'); for (Object u : users) { Map<String,Object> user = (Map<String,Object>) u; if (email.equalsIgnoreCase((String) user.get('emailAddress'))) return (String) user.get('id'); } return null; }
Why this is better: After the first time an agent is mapped (Case B or C), every subsequent call for that agent hits Case A — zero external API calls. The Salesforce User record becomes the authoritative mapping store. If an agent's Gong account changes, update Gong_Numeric_Id__c on their SF User record directly.

// Step 4: build the Gong Calls API payload Map<String,Object> payload = new Map<String,Object> { 'clientUniqueId' => conversationId, // Genesys conversationId 'actualStart' => conversationStart, // ISO 8601 'direction' => mapDirection(originatingDirection), // 'Inbound'/'Outbound' 'primaryUser' => gongNumericId, // resolved in Step 3 'downloadMediaUrl' => recordingUrl, // from Genesys Recordings API 'parties' => buildParties(participants, gongNumericId), 'context' => buildContext(accountId, opportunityId) // No phone number · no email address — PII excluded per policy }; // Context block — forces Gong to associate call with SF Account + Opportunity private static List<Object> buildContext(Id accountId, Id opportunityId) { List<Map<String,Object>> objects = new List<Map<String,Object>>(); if (opportunityId != null) objects.add(new Map<String,Object>{'objectType'=>'Opportunity','objectId'=>String.valueOf(opportunityId)}); if (accountId != null) objects.add(new Map<String,Object>{'objectType'=>'Account','objectId'=>String.valueOf(accountId)}); return new List<Object>{new Map<String,Object>{'system'=>'Salesforce','objects'=>objects}}; }

// Step 5: POST to Gong and store the returned callId on the Task record HttpRequest req = new HttpRequest(); req.setEndpoint('callout:Gong_NC/v2/calls'); req.setMethod('POST'); req.setHeader('Content-Type', 'application/json'); req.setBody(JSON.serialize(payload)); HttpResponse res = new Http().send(req); if (res.getStatusCode() == 200) { Map<String,Object> result = (Map<String,Object>) JSON.deserializeUntyped(res.getBody()); String gongCallId = (String) result.get('callId'); // Store GongCallId__c on the Task — this is what Phase 2 webhook uses Task t = new Task(Id = taskId, GongCallId__c = gongCallId); update t; } else { // Log error — do not throw, allow other calls in batch to continue logError(conversationId, 'Gong upload failed: ' + res.getBody()); }
Why store GongCallId__c on the Task: When Gong fires the Phase 2 webhook, it sends the metaData.id (Gong call ID) and the clientUniqueId. Both values are stored on the Task — Salesforce can match the incoming webhook to the right Task using either. The clientUniqueId is the primary key since it was set at call creation time.

Phase 2 · Gong webhook triggers Salesforce write-back

When Gong finishes analysing a call, an Automation Rule fires a webhook to a Salesforce public endpoint. The payload contains all needed analysis fields. Salesforce receives it, matches the Task via clientUniqueId, and updates the record natively via Apex DML.

What Gong's webhook payload contains: The webhook fires when an Automation Rule condition is met (e.g. "all processed calls"). The payload includes metaData.id, metaData.url, content.topics[], interaction.interactionStats[] (including Talk Ratio as a named stat), and the context block with Salesforce object IDs. Salesforce does not need to make a follow-up call to /v2/calls/extensive — all needed fields arrive in the webhook body.

Phase 2 · Gong Automation Rule setup (admin task)
W1
Create an Automation Rule in Gong Admin Center
Gong Admin Center → Settings → Ecosystem → Automation rules
Gong webhook

This is a one-time configuration in Gong. No code required — done through the Gong UI by a Gong Technical Admin.

  1. 1 Go to Gong Admin Center → Settings → Ecosystem → Automation Rules → click + Add Rule
  2. 2 Click the filter icon to define which calls trigger the rule. For this integration: All processed calls (no additional filters needed — every call uploaded by Phase 1 should trigger Phase 2).
  3. 3 In the Action dropdown, Fire webhook is selected by default. Set the Destination URL to the Salesforce Site endpoint (configured in the next step).
  4. 4 Authentication: select Signed JWT header. Copy the public key shown — this is used in the Apex handler to verify the request is genuinely from Gong.
  5. 5 Click Test now to verify Gong can reach the endpoint. Enable the rule once the test succeeds.
Destination URL format: https://[your-sf-site].force.com/services/apexrest/gong/webhook?token=YOUR_SECRET_TOKEN. The token in the URL is a shared secret you define — the Apex handler checks it before processing the payload, as a first layer of authentication on top of JWT verification.
Phase 2 · Salesforce public endpoint — Apex REST resource
W2
Create an Apex REST class to receive the Gong webhook POST
Exposed via Salesforce Site — public URL, no Salesforce login required
Apex REST

Salesforce exposes a public HTTPS endpoint by creating an Apex REST Resource and exposing it through a Salesforce Site (or Experience Cloud site). Gong POSTs its webhook payload to this URL — no Salesforce authentication required from Gong's side.

Salesforce Site setup: Setup → Sites → New Site → set the site URL prefix → under Public Access Settings, grant the Guest User permission to access the Apex REST class. The resulting URL will be https://[yourorg].my.site.com/services/apexrest/gong/webhook.
// GongWebhookResource.apex — public REST endpoint for Gong webhooks @RestResource(urlMapping='/gong/webhook/*') global class GongWebhookResource { @HttpPost global static void handlePost() { RestRequest req = RestContext.request; RestResponse res = RestContext.response; // 1. Verify shared token in query param as first-line security check String token = req.params.get('token'); if (token != System.Label.Gong_Webhook_Token) { res.statusCode = 401; return; } // 2. Parse the Gong webhook payload String body = req.requestBody.toString(); Map<String,Object> payload = (Map<String,Object>) JSON.deserializeUntyped(body); // 3. Hand off to async handler to avoid governor limit issues // Using @future so the REST response returns immediately to Gong GongWebhookHandler.processAsync(body); // 4. Respond 200 immediately — Gong will retry if it doesn't get 2xx res.statusCode = 200; res.responseBody = Blob.valueOf('{"status":"received"}'); } }
Always return 200 immediately. Gong will retry the webhook if it does not receive a 2xx response quickly. Use @future or a Queueable to do the actual processing asynchronously — respond to Gong first, process after.

// GongWebhookHandler.apex — processes the Gong webhook payload asynchronously public class GongWebhookHandler { @future public static void processAsync(String rawBody) { try { Map<String,Object> payload = (Map<String,Object>) JSON.deserializeUntyped(rawBody); Map<String,Object> callData = (Map<String,Object>) payload.get('callData'); if (callData == null) return; // safety check // ── Extract fields from webhook payload ────────────────────────────────── // No second API call to /v2/calls/extensive needed — // Gong's webhook payload already contains topics, talk ratio, and call URL Map<String,Object> metaData = (Map<String,Object>) callData.get('metaData'); Map<String,Object> content = (Map<String,Object>) callData.get('content'); Map<String,Object> interaction = (Map<String,Object>) callData.get('interaction'); String gongCallId = (String) metaData.get('id'); String gongCallUrl = (String) metaData.get('url'); String clientUniqueId = (String) metaData.get('clientUniqueId'); // Extract topics as comma-joined string List<Object> topicsList = (List<Object>) content?.get('topics'); String topics = joinTopics(topicsList); // Extract talk ratio from interactionStats — Gong names it "Talk Ratio" List<Object> stats = (List<Object>) interaction?.get('interactionStats'); String talkRatio = extractStat(stats, 'Talk Ratio'); // ── Match the Salesforce Task via clientUniqueId ────────────────────────── // No external storage needed — Task is queryable directly in Salesforce List<Task> tasks = [ SELECT Id, GongAnalysisSyncStatus__c FROM Task WHERE clientUniqueId__c = :clientUniqueId LIMIT 1 ]; if (tasks.isEmpty()) { logError(gongCallId, 'No Task found for clientUniqueId: ' + clientUniqueId); return; } // ── Prevent duplicate processing ───────────────────────────────────────── if ('Synced'.equals(tasks[0].GongAnalysisSyncStatus__c)) return; // ── DML update — native Apex, no REST API call to Salesforce ───────────── Task t = new Task( Id = tasks[0].Id, GongCallId__c = gongCallId, GongCallUrl__c = gongCallUrl, GongTopics__c = topics, GongTalkRatio__c = talkRatio, GongAnalysisSyncStatus__c = 'Synced', GongAnalysisSyncedAt__c = Datetime.now() ); update t; } catch (Exception e) { logError('webhook', e.getMessage()); } } // Helper: join topic names from Gong's topics array private static String joinTopics(List<Object> topicsList) { if (topicsList == null) return ''; List<String> names = new List<String>(); for (Object t : topicsList) { Map<String,Object> topic = (Map<String,Object>) t; names.add((String) topic.get('name')); } return String.join(names, ', '); } // Helper: extract named stat from interactionStats array private static String extractStat(List<Object> stats, String statName) { if (stats == null) return null; for (Object s : stats) { Map<String,Object> stat = (Map<String,Object>) s; if (statName.equals(stat.get('name'))) return String.valueOf(stat.get('value')); } return null; } }

Governor limits — what to watch for in the POC

Both phases run inside Salesforce transactions. The key limits are callouts per transaction (100 max) and the requirement to keep callouts out of loops. Phase 2 webhook is one callout-free transaction.

LimitValuePhase 1 impactPhase 2 impactSafeguard
HTTP callouts per transaction 100 ~4 callouts per call (Genesys analytics, recordings, Gong users if needed, Gong upload) Zero — webhook is inbound, no callouts in handler Process 20 calls max per Phase 1 Apex run for POC
Callout timeout total 120s 4 calls × 3s each = well within limit N/A Set individual callout timeout to 20s
SOQL queries per transaction 100 ~3 queries per call (Task, Contact, User) 1 query (Task by clientUniqueId) No issue for POC volumes
DML statements 150 1–2 (Task update, User update if remapping) 1 (Task update) No issue
@future callouts Not allowed N/A Phase 2 handler uses @future — cannot make callouts from @future Phase 2 webhook handler does NOT need outbound calls — payload is self-contained ✓
POC volume cap: Process a maximum of 5–20 call records per Phase 1 Scheduled Apex run. Filter by a narrow time window and explicitly skip Tasks where GongCallId__c is already populated. For full production scale, Phase 1 needs a Queueable Apex chain pattern to handle bulk volumes.
